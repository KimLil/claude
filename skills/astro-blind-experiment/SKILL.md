# Слепой астрологический эксперимент

## Обзор процесса

Цель эксперимента — дать человеку четыре обезличенных психологических описания,
основанных на натальных картах четырёх дат (истинная + три смещения), и предложить
оценить каждое без знания, какая дата является настоящей. Это проверяет, насколько
точно астрологическая интерпретация описывает личность.

Все восемь профилей — это самостоятельные психологические описания. Никаких
технических астрологических терминов в тексте профилей.

---

## Шаг 1 — Первый ответ: запросить данные

**В первом ответе только:**
1. Подтвердить, что инструкции приняты.
2. Запросить:
   - дату рождения (день, месяц, год);
   - точное время рождения (часы и минуты);
   - место рождения (город, страна).

Не начинать генерацию до получения всех трёх данных.

---

## Шаг 2 — Вычислить даты и создать папку

После получения данных выполнить в bash:

```bash
python3 - <<'EOF'
from datetime import date, timedelta

# Подставить данные пользователя
bd = date(YYYY, MM, DD)  # заменить на реальные значения

dates = {
    "BD":     bd,
    "BD+90":  bd + timedelta(days=90),
    "BD-90":  bd - timedelta(days=90),
    "BD+180": bd + timedelta(days=180),
}

for key, d in dates.items():
    print(f"{key}: {d.strftime('%d.%m.%Y')} ({d.strftime('%Y-%m-%d')})")
EOF
```

**Создать выходную папку** внутри папки проекта пользователя:
`bd_{YYYYMMDD}_{HHMM}/`  — например `bd_19880101_1835/`

Все файлы сессии сохранять только в эту папку.

---

## Шаг 3 — Определить астрологические сигнатуры

Для каждой из 4 дат определить:

**Western (тропический зодиак):**
- Sun sign по таблице в `references/western_archetypes.md`
- Примерный Moon sign: Луна проходит ~13°/день; определить фазу Луны относительно BD
  и оценить знак. Точность ±1 знак приемлема — важнее психологический архетип, чем
  астрономическая точность.
- Время и место влияют на Асцендент — учесть это при описании темперамента.

**Jyotish (сидерический зодиак, аянамша ~23–24°):**
- Вычесть ~23 дня из тропической даты для получения сидерического Sun sign
- Определить накшатру Солнца (см. `references/jyotish_archetypes.md`)
- Правитель накшатры модифицирует характер знака

Прочитать соответствующие разделы в reference-файлах **перед написанием профилей**.

---

## Шаг 4 — Написать профили западной астрологии

Написать 4 профиля: west_BD.txt, west_BD+90.txt, west_BD-90.txt, west_BD+180.txt

**Требования к каждому профилю:**
- Объём: ~1000 слов, на русском языке
- Структура: см. раздел «Формат профиля» ниже
- Стиль: независимое психологическое описание, без астрологических терминов
- Профили должны заметно отличаться друг от друга
- Нет прогнозов, советов, описания событий жизни
- Акцент на устойчивых чертах характера

Сохранять каждый профиль в файл сразу после написания.

---

## Шаг 5 — Написать профили Джйотиш

Написать 4 профиля: djo_BD.txt, djo_BD+90.txt, djo_BD-90.txt, djo_BD+180.txt

**Требования те же, плюс:**
- Генерировать независимо от западных профилей — не переписывать, не перефразировать
- Опираться на Джйотиш-традицию: акцент на дхарме, карме, ролях, служении, качестве
  ума (манас), природе действия (раджас/тамас/саттва) — но без этих санскритских слов
  в тексте профиля
- Совпадения с западными профилями допустимы, только если обе традиции действительно
  приводят к схожим выводам

---

## Шаг 6 — Рандомизация

После написания всех 8 профилей выполнить скрипт рандомизации:

```bash
python3 /path/to/skill/scripts/randomize.py /path/to/output/folder
```

Скрипт:
1. Случайно перемешивает западные профили → назначает метки A–D
2. Случайно перемешивает Джйотиш профили → назначает метки K–N
3. Проверяет отсутствие позиционных совпадений между школами (ни один индекс позиции
   не должен содержать одинаковую дату-суффикс в обеих школах)
4. Сохраняет `mapping.json` в папку

Показать пользователю подтверждение рандомизации.

## Шаг 7 — Публикация результатов через Artifacts

Вместо генерации HTML-файлов результаты публикуются прямо в чат через инструмент `show_widget`.

### 7.1 — Прочитать все данные из папки

```bash
python3 - <<'EOF'
import os, json

folder = "/path/to/output/folder"  # заменить на реальный путь

with open(os.path.join(folder, "mapping.json"), encoding="utf-8") as f:
    mapping = json.load(f)

birth_info_path = os.path.join(folder, "birth_info.json")
if os.path.exists(birth_info_path):
    with open(birth_info_path, encoding="utf-8") as f:
        birth_info = json.load(f)
else:
    birth_info = {}

print("=== MAPPING ===")
print(json.dumps(mapping, ensure_ascii=False, indent=2))

all_maps = {**mapping["west"], **mapping["djo"]}
for label in sorted(all_maps.keys()):
    fname = all_maps[label]
    path = os.path.join(folder, fname + ".txt")
    text = open(path, encoding="utf-8").read()
    print(f"\n=== ПРОФИЛЬ {label} ({fname}) ===")
    print(text)
EOF
```

### 7.2 — Вызвать read_me, затем show_widget четыре раза

Сначала вызвать `visualize:read_me` с модулем `interactive`.

Затем вызвать `show_widget` четыре раза — **поочерёдно**, не подряд без текста между ними.
Перед каждым виджетом добавить короткую фразу-пояснение (например: «Тест западной школы готов:»).

**Artifact 1 — Тест западной школы** (title: `astro_test_west`)

HTML-страница с профилями A, B, C, D. Структура каждой карточки:
- Заголовок «Профиль A» и т.д.
- Шкала «Насколько это похоже на меня» (радиокнопки 1–10)
- Чекбокс «Это точно про меня!»
- Сворачиваемые секции `<details>` для каждого раздела профиля;
  только «КРАТКОЕ РЕЗЮМЕ ЛИЧНОСТИ» открыта по умолчанию
- Кнопка «Скачать результаты (JSON)» внизу страницы — через Blob API

Тексты профилей встраиваются **прямо в HTML**, не читаются с диска.

**Artifact 2 — Ключ западной школы** (title: `astro_key_west`)

HTML-страница:
- Предупреждение: «Открывать только после оценки всех профилей»
- Кнопка «Посмотреть ключ» — раскрывает скрытый блок (`display:none → block`)
- Таблица: Профиль → Дата → Тип варианта; истинная дата выделена

**Artifact 3 — Тест Джйотиш** (title: `astro_test_djo`)

Аналогично Artifact 1, профили K, L, M, N.

**Artifact 4 — Ключ Джйотиш** (title: `astro_key_djo`)

Аналогично Artifact 2.

### 7.3 — Кнопка скачивания JSON

`localStorage` и `sessionStorage` **не работают** в Artifacts. Использовать Blob API:

```javascript
function downloadResults(school) {
    const results = {};
    document.querySelectorAll('.profile-card').forEach(card => {
        const label = card.dataset.label;
        const rating = card.querySelector('input[type=radio]:checked')?.value || null;
        const exact = card.querySelector('input[type=checkbox]')?.checked || false;
        results[label] = { rating, exact };
    });
    const blob = new Blob(
        [JSON.stringify({ school, results }, null, 2)],
        { type: 'application/json' }
    );
    const a = document.createElement('a');
    a.href = URL.createObjectURL(blob);
    a.download = `astro_${school}_results.json`;
    a.click();
}
```

## Шаг 8 — Контроль качества

Перед публикацией Artifacts проверить:

1. Все 8 TXT-файлов существуют и не пустые
2. Рандомизация выполнена: mapping.json существует
3. Ключи соответствуют файлам
4. Школы не смешаны (west_ → A–D, djo_ → K–N)
5. В профилях отсутствуют запрещённые термины (см. список ниже)
6. Профили заметно различаются между собой

```bash
python3 - <<'EOF'
import os, json, glob

folder = "/path/to/output/folder"

# Проверить TXT-файлы
txt_files = ["west_BD", "west_BD+90", "west_BD-90", "west_BD+180",
             "djo_BD",  "djo_BD+90",  "djo_BD-90",  "djo_BD+180"]
for f in txt_files:
    path = os.path.join(folder, f + ".txt")
    size = os.path.getsize(path) if os.path.exists(path) else 0
    print(f"{'✓' if size > 5000 else '✗'} {f}.txt ({size} bytes)")

# Проверить mapping
mapping_path = os.path.join(folder, "mapping.json")
if os.path.exists(mapping_path):
    m = json.load(open(mapping_path))
    print(f"✓ mapping.json: west={m['west']}, djo={m['djo']}")
else:
    print("✗ mapping.json НЕ НАЙДЕН")
EOF
```

После успешной проверки — опубликовать четыре Artifacts согласно Шагу 7.
Сообщить пользователю кратко: что показано, в каком порядке смотреть (сначала тест, потом ключ).

## Запрещённые термины в профилях

Ни в одном профиле НЕ использовать:
- названия знаков зодиака (Козерог, Овен, Весы и др.)
- названия планет как астрологических факторов (Марс, Венера и др.)
- Асцендент, Асц, ASC
- Дома (1-й дом, 7-й дом и т.д.)
- Накшатры, Раху, Кету, Лагна
- Аянамша, аспекты, соединения, оппозиции
- Любые прямые намёки на дату
